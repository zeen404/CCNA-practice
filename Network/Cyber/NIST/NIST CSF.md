# NIST Cybersecurity Framework (CSF) 2.0

![[Pasted image 20260430131519.png]]

> [!abstract] ภาพรวมของ NIST CSF
> กรอบงานความมั่นคงปลอดภัยไซเบอร์ (NIST CSF) เป็นแนวทางสากลที่ช่วยให้องค์กรทุกขนาดสามารถจัดการและลดความเสี่ยงด้านไซเบอร์ได้อย่างมีประสิทธิภาพ

---

## 🕒 วิวัฒนาการของ NIST CSF

| เวอร์ชัน | ปีที่เผยแพร่ | จุดเน้นสำคัญ                                              |
| :------- | :----------- | :-------------------------------------------------------- |
| **v1.0** | 2014         | การระบุ ประเมิน และจัดการความเสี่ยงเบื้องต้น              |
| **v1.1** | 2018         | ปรับปรุงเนื้อหาให้ชัดเจนและใช้งานง่ายขึ้น                 |
| **v2.0** | 2023         | เพิ่มฟังก์ชัน **Govern (การกำกับดูแล)** เพื่อความครอบคลุม |

## 🛠️ องค์กรหลักของ NIST CSF 2.0 (The Core)

ในเวอร์ชัน 2.0 ได้มีการเพิ่มฟังก์ชันการกำกับดูแลเข้ามา ทำให้ปัจจุบันประกอบด้วย **6 ฟังก์ชันหลัก**:

### ⚖️ 1. การกำกับดูแล (Govern)
การปฏิบัติตามหน้าที่ด้านการกำกับดูแลจำเป็นต้องมีความเข้าใจบริบทขององค์กร กำหนดระดับความเสี่ยงที่ยอมรับได้ (**Risk Appetite**) และกำหนดบทบาทหน้าที่ความรับผิดชอบอย่างชัดเจน
- **ความท้าทาย:** ความซับซ้อนในการบริหารจัดการให้สอดคล้องกับวัตถุประสงค์องค์กร และการจัดการความเสี่ยงจาก **ผู้ให้บริการภายนอก (Third-party Risk)**
- **เป้าหมาย:** ส่งเสริมความรับผิดชอบและผลักดันให้เกิดการปรับปรุงอย่างต่อเนื่องผ่านมาตรการควบคุมที่มีประสิทธิภาพ

### 🔍 2. การระบุความเสี่ยง (Identify)
การจัดทำรายงานเพื่อระบุความเสี่ยงของอุปกรณ์และสินทรัพย์ต่างๆ โดยจัดลำดับตามความสำคัญ
- **กระบวนการ ALM:** มีกระบวนการจัดการสินทรัพย์ตั้งแต่เริ่มต้นจนหมดอายุการใช้งาน (**Asset Lifecycle Management**)
- **ข่าวกรอง:** ตรวจสอบและรวบรวมข่าวกรองด้านภัยคุกคามเพื่อใช้ประเมินความเสี่ยงในปัจจุบัน

### 🛡️ 3. การป้องกัน (Protect)
การใช้มาตรการรักษาความปลอดภัยเพื่อปกป้องระบบการทำงานและข้อมูลขององค์กร
- **มาตรการ:** กำหนดสิทธิการเข้าถึง (**Identity & Access Management**), การใช้เทคโนโลยีป้องกันระบบ
- **บุคลากร:** จัดอบรมเพื่อสร้างความตระหนักรู้ (**Security Awareness Training**) เกี่ยวกับภัยคุกคามทางไซเบอร์

### 🚨 4. การตรวจจับ (Detect)
การติดตามสินทรัพย์อย่างต่อเนื่องเพื่อตรวจจับสิ่งผิดปกติและสัญญาณบ่งชี้ถึงภัยคุกคาม
- **การวิเคราะห์:** มีความสามารถในการวิเคราะห์เหตุการณ์ที่น่าสงสัยได้อย่างรวดเร็ว
- **เป้าหมาย:** ระบุลักษณะและตรวจพบพฤติกรรมที่ผิดปกติของระบบ เพื่อให้ตอบสนองและแก้ไขปัญหาได้อย่างทันท่วงที ลดผลกระทบที่อาจเกิดขึ้น

### 📞 5. การรับมือ (Respond)
การดำเนินการเมื่อตรวจพบเหตุการณ์ผิดปกติ
- **กิจกรรม:** การวางแผนรับมือเหตุการณ์ (Incident Response Plan), การสื่อสารระหว่างเกิดเหตุ

### 🔄 6. การกู้คืน (Recover)
การทำให้ระบบกลับมาทำงานได้ตามปกติหลังจากถูกโจมตี
- **กิจกรรม:** การกู้คืนข้อมูลสำรอง, การสรุปบทเรียนเพื่อปรับปรุงแผนในอนาคต์

---

> [!important] สรุปการเปลี่ยนแปลงใน v2.0
> การเพิ่ม **Govern** เข้ามาช่วยให้การตัดสินใจด้านความมั่นคงปลอดภัยไซเบอร์เชื่อมโยงกับกลยุทธ์ทางธุรกิจและลำดับความสำคัญขององค์กรได้ดียิ่งขึ้น
